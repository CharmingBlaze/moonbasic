DUPLICATES (0-C) — same behavior without clear alias comment, or dual implementations
================================================================================
1) PRINT / PRINTLN — both always emit newline via Fprintln (no PRINTLN-specific behavior).
2) SQR / SQRT — intentional sqrt aliases (MATH.* mirrors).
3) ATN / ATAN — both atan (BASIC compat + modern name); intentional.
4) ARRAYFREE / ERASE — same handler (intentional).
5) ARRAYJOINS / ARRAYJOINS$ — same handler (intentional).
6) strmod.* vs core — strmod registers strmod.LSET$, strmod.FORMAT$, etc. (not manifest keys); parallel logic to core for COUNT$/IS*/FORMAT$ with different STRING$/ISNUMERIC rules — consolidate to one canonical Register per name.
7) core.BAND vs intended BAND — wrong registry key; should be thin alias BAND → same fn as core.BAND or rename registration.
8) UTIL.* vs spec FILE*/GET* names — same intended surface, different command keys (needs alias layer).

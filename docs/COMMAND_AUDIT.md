## How documentation is organized

1. **`docs/COMMANDS.md`** — Topic index, human-written explanations, DONE/PARTIAL status, and tables for major modules.
2. **`docs/reference/*.md`** — Deep dives per subsystem (window, render, physics, …).
3. **`docs/API_CONSISTENCY.md`** — Generated list of every manifest overload and arity (`go run ./tools/apidoc`).
4. **`docs/reference/API_CONVENTIONS.md`** — Recommended verbs (`LOAD`, `SETPOS`, …) across object types.
5. **Optional `description` fields** in `commands.json` — Extra inline help for tools; not required for every row.
